§ I  ·  Methodology

A test that changes the network, then asks the network to fix itself.

Most pipeline tests assert that an API returns 200. This one asserts that a configuration drift, induced over SSH on a production-style router, is detected by the model, classified, queued for approval, executed by pyATS, re-verified by a second snapshot, and confirmed by the device itself — all unattended, end-to-end.

i.

Pre-flight

Five gates: API health, dependency mesh, ICMP reachability of each lab target, and a verified golden baseline snapshot on file for every device under test.

ii.

Safety rails

Every config push is screened for management-subnet references and a list of lockout-inducing tokens. Reachability is checked before and after each push; the harness aborts loud if the device falls silent.

iii.

Inject

The harness opens an SSH session, applies the drift, and writes memory. The lab is now in an unsanctioned state, identical to one a junior engineer might leave behind.

iv.

Observe & act

Parity takes a fresh snapshot, reasons over the diff with Gemini 2.5 Flash, queues an approval, executes the remediation via pyATS, re-snapshots to verify, and resolves the incident. The harness then queries the device directly to confirm.

§ V  ·  A discovery, candidly reported

The agent refused to guess.

A prefix-list filter on DC2-R2 was detected — but not auto-remediated. By design.

The original Scenario C applied an inbound prefix-list to a BGP neighbour, silently filtering one learned route. Parity's snapshot captures BGP operational state — sessions, prefix counts, capabilities — but not the policy attachments themselves; that data lives one layer down, in the running configuration.

The reasoner saw the symptom (a route had vanished) and refused to draft remediation commands without seeing the cause. This is the correct behaviour: an autonomous agent that hallucinates fixes for symptoms it does not understand is the failure mode every CIO loses sleep over. Parity stopped short, raised the finding for human attention, and waited.

We replaced Scenario C with a static-route drift — a class the snapshot does fully cover — and proceeded with the demonstration. The original case is filed as a follow-up: extend snapshot capture for BGP neighbour policy. The honest tooling moment is, in many ways, the most reassuring result in this report.
